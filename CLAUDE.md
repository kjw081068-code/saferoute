# 프로젝트 규칙

## 언어
- **반드시** 한국어로만 응답할 것
- 영어로 질문받아도 한국어로 답변
- 코드 주석도 한국어로 작성
- 어떤 상황에서도 영어 응답 금지

## 작업 시작 전
- 모든 작업 요청을 받으면 바로 실행하지 말고
  "Plan Mode로 먼저 계획을 확인할까요?" 라고 물어볼 것
- 사용자가 승인하면 계획을 보여주고, 승인 후 실행

## 작업 완료 후
- 작업이 끝나면 항상 마지막에 /cost 명령어를 실행해서
  현재 세션의 토큰 사용량과 비용을 알려줄 것

  ## 코딩 규칙
- 파일 수정 전 반드시 현재 파일 내용 먼저 확인
- 기존 코드 스타일 유지 (임의로 리팩토링 금지)
- 한 번에 여러 파일 대규모 수정 금지 → 하나씩 확인 후 진행

## 오류 처리
- 에러 발생 시 원인 한국어로 설명 후 수정
- 확실하지 않은 건 추측으로 수정하지 말고 먼저 물어볼 것

## 커뮤니케이션
- 작업 시작 전 "무엇을 할 것인지" 한 줄 요약
- 작업 중 진행상황 단계별로 알려주기
- 모르거나 애매하면 추측하지 말고 질문할 것

## 비전공자 배려
- 코드 작성 후 "이 코드가 하는 일"을 한국어로 2~3줄 요약
- 생소한 개념이 나오면 쉬운 말로 부연 설명
- 왜 이 방법을 선택했는지 간단히 이유 설명

## 플러그인 사용 규칙

### Context7
- 라이브러리, 프레임워크 관련 코드 작성 시 항상 context7 사용
- 프롬프트에 "use context7"을 자동으로 포함해서 실행
- Next.js, React, Tailwind, Supabase 등 외부 라이브러리는
  반드시 context7으로 최신 문서 확인 후 코드 작성

### Feature-dev
- 새로운 기능 개발 요청이 오면 바로 코드 짜지 말고
  /feature-dev 워크플로우로 먼저 설계 단계 진행
- "~만들어줘", "~추가해줘", "~구현해줘" 같은 요청은
  feature-dev 대상으로 판단

  ## 터미널 출력 처리
- bash 명령 실행

## 터미널 출력 처리
- bash 명령 실행 후 출력 결과를 항상 한국어로 해석해서 설명
- 에러 메시지가 나오면 원문 그대로 두고 아래에 한국어 설명 추가
- 성공/실패 여부를 한국어로 요약
